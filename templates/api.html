API

The API allows you to query our submission database for the list of the categorized problems. You'll get them anonymized and without a timestamp, the format is JSON, which should be digestible in almost any programming language.

Select a problem class below for an example:

{% include "problem_selector.shtml" %}

Query parameters

The basic URL for GET requests is api. We support/require the following parameters:

  • release The name of the release, e.g. "r2.1i"
  • problem The name of the problem as it appear in the problem selector (URL encoding is required, e.g. whitespace becomes %20)

Other queries?

The set of queries we currently allow is restricted to global queries based on the problem class. If you want to be able to query our database differently, tell us by opening a GitHub issue.
